Tailwind - Alpine Exercises

1: Real-time Name Display ▴ ▾ (click to expand)

Exercise: ./name_display.html

Preview: ./preview/name_display.mp4

Part 1: Tailwind CSS Styling

  1. Body:
    • Set a dark cyan background color for the entire page.
    • Add padding around the edges.
    • Make the body take up at least the full height of the viewport.
    • Center the content both vertically and horizontally.
  2. Button:
    • Position the button absolutely at the top of the page, centered horizontally.
    • Style it with a cyan background color and a darker cyan border.
    • Add padding, rounded corners, and margin.
  3. Main container:
    • Create a white background container with a shadow and rounded corners.
    • Set maximum width and full width within that maximum.
    • Add generous padding inside the container.
  4. Heading:
    • Style the heading with larger text, bold font, centered alignment, and dark gray color.
    • Add some margin at the bottom.
  5. Label:
    • Make the label text larger and semi-bold with a dark gray color.
  6. Input field:
    • Add a border and style it to change color when focused.
    • Include padding, rounded corners, and take up the full width.
    • Implement a smooth transition effect for focus state.
  7. Display paragraph:
    • Center the text and use a medium-large font size.
    • Style the name display span with bold text and blue color.

Part 2: Alpine.js Functionality

  1. Visibility toggle:
    • Create a data property isVisible initialized to false.
    • Toggle the visibility of the main container when the button is clicked.
    • Change the button text between "Show" and "Hide" based on the visibility state.
  2. Name input and display:
    • Create a data property name initialized to an empty string.
    • Bind the input field to the name property using x-model.
    • Display the entered name in real-time.
    • Show a placeholder text when no name is entered.
  3. Cloak directive:
    • Use the x-cloak directive to prevent any flickering of Alpine.js template expressions during page load.

2: Post-It ▴ ▾ (click to expand)

Exercise: ./postit.html

Preview: ./preview/pistit.mp4

  1. Body:
    • Apply a light gray background color to the entire page.
  2. Main Container:
    • Center the content and add a top margin.
  3. h1 Tag:
    • Style the text to be larger, bolder, centered, and dark red in color.
  4. Horizontal Rule:
    • Add vertical spacing above and below the horizontal rule.
  5. Content Grid:
    • Design a responsive grid layout that transitions from 1 column on small screens to 2 columns on medium screens and 3 columns on larger screens.
    • Include horizontal padding on small screens, removing it on medium and larger screens.
    • Use a vertical gradient background from white to light gray.
    • Add a dark red border and rounded corners to the grid container.
    • Apply a shadow effect to the container.
    • Include padding inside the container.
  6. Paragraphs within the Grid:
    • Style the text in dark blue.
    • Change the text color to pink on hover.
    • Set the cursor to "not-allowed" on hover.
    • Add a border and padding to each paragraph.
    • Apply a shadow effect that intensifies on hover.
    • Implement smooth transitions for hover effects.
  7. Individual Paragraph Hover Effects:
    • First Paragraph: Rotate 3 degrees counter-clockwise and change the background to light green on hover.
    • Second Paragraph: Rotate 3 degrees clockwise and change the background to light amber on hover.
    • Third Paragraph: Rotate 6 degrees counter-clockwise and change the background to light blue on hover.
    • Fourth Paragraph: Rotate 3 degrees clockwise and change the background to light amber on hover.
    • Fifth Paragraph: Rotate 15 degrees clockwise and change the background to light lime on hover.
    • Sixth Paragraph: Rotate 3 degrees clockwise and change the background to light amber on hover.
    • Seventh Paragraph: Rotate 5 degrees counter-clockwise and change the background to light orange on hover.
    • Eighth Paragraph: Rotate 3 degrees clockwise and change the background to light teal on hover.

3: Todo List ▴ ▾ (click to expand)

Exercise: ./todo.html

Preview: ./todo.mp4

Part 1: Tailwind CSS Styling

  1. Body:
    • Set a dark blue background color for the entire page.
    • Make the body take up the full height of the viewport.
    • Center the content both vertically and horizontally.
    • Add padding around the edges.
  2. Main container:
    • Create a white background container with rounded corners.
    • Set maximum width and center it horizontally.
    • Add padding inside the container.
  3. Heading:
    • Style the heading with larger text, bold font, centered alignment.
    • Add padding around the heading.
  4. Search input:
    • Style the input with full width, padding, border, and rounded corners.
    • Add vertical margin.
  5. Todo list:
    • Create space between list items.
  6. List items:
    • Style with light blue background, padding, and rounded corners.
    • Set text color to dark blue.
    • On hover:
      • Change background to dark blue and text to white.
      • Add a dark blue border.
      • Increase padding slightly.
      • Change cursor to pointer.
    • Implement smooth transitions for hover effects.
  7. Add new item input:
    • Style similar to the search input.
    • Add some margin at the bottom.
  8. Add button:
    • Style with dark blue background and white text.
    • Add padding, rounded corners, and make it full width.

Part 2: Alpine.js Functionality

  1. Data management:
    • Create data properties for search, newItem, and items (initialized with some default values).
  2. Computed property:
    • Implement a filteredItems computed property that filters the items based on the search input.
  3. Add item functionality:
    • Create an addItem method that adds the newItem to the items array and clears the input.
  4. Delete item functionality:
    • Implement a deleteItem method that removes the clicked item from the items array.
  5. Search functionality:
    • Bind the search input to the search data property.
    • Use the filteredItems computed property to display the list.
  6. List rendering:
    • Use x-for directive to render the filtered list items.
    • Bind the item text and click event for deletion.
  7. New item input:
    • Bind the input to the newItem data property.
    • Trigger the addItem method when the Add button is clicked.
